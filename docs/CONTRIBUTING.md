# Contributing Guide

## Branch Naming
- `feature/<name>` — new features
- `fix/<name>` — bug fixes
- `chore/<name>` — deps, config, refactors

## Commit Convention
See `.github/COMMIT_CONVENTION.md`

## PR Process
1. Branch from `develop`
2. Open PR against `develop`
3. All checks must pass
4. Squash merge preferred
